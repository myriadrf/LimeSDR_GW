# LimeSDR_GW Tools

> [!IMPORTANT]
> These tools are intended for **internal development use**. As such, they may be experimental, contain bugs, or be less user-friendly than production-grade software. Use with caution.

This directory contains utility scripts and GNU Radio flowgraphs designed to assist in the development, debugging, and maintenance of the LimeSDR Gateware.

## Python Utility Scripts

The scripts below are grouped by their primary function within the development lifecycle.

### 1. Project Auditing & Visualization
Tools used to maintain a clean codebase and visualize project dependencies.

| Script | Function Description | Usage Example | Use Case |
| :--- | :--- | :--- | :--- |
| **`project_auditor.py`** | Scans the project tree using mock imports and build functions to detect HDL (Verilog/VHDL) and Python source files that are not referenced in the build system. | `python3 tools/project_auditor.py` | Identifying and removing "dead" code to reduce project bloat and improve build times. |
| **`audit_gui.py`** | A Streamlit web application that provides a graphical interface for the results generated by `project_auditor.py`, allowing for interactive exploration of file dependencies. | `streamlit run tools/audit_gui.py` | Visual identification of unused components and navigating the complex project structure. |

### 2. Build & Resource Analysis
Utilities for analyzing the results of LiteX builds and optimizing FPGA resource utilization.

| Script | Function Description | Usage Example | Use Case |
| :--- | :--- | :--- | :--- |
| **`mem_analyzer.py`** | Parses ELF and MAP files from firmware builds to report memory region occupancy (BIOS, SRAM, RAM) and provides recommendations for optimal memory sizing. | `python3 tools/mem_analyzer.py <file.elf>` | Calculating the minimum required memory sizes to save FPGA BRAM resources while ensuring firmware stability. |
| **`target_info.py`** | Mocks the `argparse` module to extract SoC metadata (CSR addresses, constants, and memory maps) from LiteX target scripts into a JSON format. | *Internal utility* | Used by build_gui.py to extract build arguments from build scripts.
| **`patches.py`** | A library of monkey-patches for LiteX and Migen that allows fine-grained control over Verilog generation by injecting Verilog attributes into memory declarations. | *Imported as a library* | Forcing the use of distributed RAM (LUTRAM) for specific small buffers that LiteX might otherwise put in Block RAM. |
| **`litex_manager.py`** | Manages LiteX environment variables, repository paths, and dependency versions. Supports cloning, installing in editable mode, and "freezing" dependencies to a config file. | *Internal utility* | Keeping developer environments synchronized with the specific LiteX/Migen versions used by the project. |

### 3. Maintenance & Validation
Tools for ensuring code quality and consistency across different hardware targets.

| Script | Function Description | Usage Example | Use Case |
| :--- | :--- | :--- | :--- |
| **`check_bsp.py`** | Scans Board Support Package (BSP) C files for placeholders like `#error`, `TODO`, or unimplemented functions. Provides both CSV and GUI (Tkinter) summaries. | `python3 tools/check_bsp.py <bsp_dir>` | Quick health check of firmware portability when porting to a new hardware platform or tracking feature completeness. |
| **`validate_regremap.py`** | Verifies the consistency of the register remapping logic between legacy LMS64C protocol (C source) and modern LiteX CSRs (CSV documentation). | `python3 tools/validate_regremap.py` | Ensuring that firmware refactors do not break compatibility with existing host-side software like LimeSuite. |
| **`generate_master_reg_list.py`** | Runs `make regmap-all` in the `docs` directory and aggregates register definitions from all board targets into a single master CSV list (`master_registers.csv`). | `python3 tools/generate_master_reg_list.py` | Generating unified register map documentation and identifying unused registers across the entire LimeSDR product family. |

### 4. Board-Specific Utilities
Scripts tailored for specific LimeSDR hardware variants.

| Script | Function Description | Usage Example | Use Case |
| :--- | :--- | :--- | :--- |
| **`limesdr_mini_v2_bitstream.py`** | Automates the post-processing of bitstreams for the Mini V2, including MCS file generation for SPI Flash and directory organization. | *Internal utility* | Preparing production-ready firmware packages for distribution and flashing. |
| **`test_gpsdo_cli.py`** | A CLI tool that interfaces with the `limeCSR` utility to monitor and tune the GPSDO PID loop in real-time. Supports check, reset, enable, and disable modes. | `python3 tools/test_gpsdo_cli.py --check` | On-bench debugging of frequency stability, GNSS lock status, and regulation loop performance. |

## GNU Radio Companion Flowgraphs

| File | Target Hardware | Description |
| :--- | :--- | :--- |
| **`test_miniv2_tx.grc`** | LimeSDR Mini V2 | Validates the TX signal path by transmitting a known waveform and checking for signal integrity. |
| **`test_xtrx_rx_null.grc`** | LimeSDR XTRX | A minimal RX flowgraph used to verify data streaming over PCIe interface by receiving into a null sink. |
